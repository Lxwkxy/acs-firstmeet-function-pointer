# 🚀 ACS First Meet: Function & Pointer 101

ยินดีต้อนรับน้องๆ สู่โลกของการเขียนโปรแกรมภาษา C! Repository นี้รวบรวมโค้ดตัวอย่างและสื่อการสอน เพื่อช่วยตกผลึกความเข้าใจในเรื่อง **Function** และ **Pointer** แบบ Step-by-Step

## 📑 สารบัญเนื้อหา (Outline)

1. **ฟังก์ชันคืออะไร (Function Basics):** แกะกล่องสมการคณิตศาสตร์ $f(x) = x + 5$ สู่โค้ดภาษา C และข้อดีของการลดโค้ดซ้ำซ้อน
2. **ชนิดของการส่งค่า (Return Type vs Void):** ความแตกต่างระหว่างเครื่องจักรที่คายผลลัพธ์ กับเครื่องจักรที่ทำงานจบในตัว
3. **ข้อมูลที่ฟังก์ชันใช้งาน:** ทำความรู้จัก Parameter (ตัวแปรที่เปิดรอรับ) และ Argument (ข้อมูลดิบที่โยนเข้าไป)
4. **ขอบเขตตัวแปร (Variable Scope):** - `Local Variable`: เกิดในฟังก์ชัน ตายในฟังก์ชัน
   - `Global Variable`: ตัวแปรส่วนรวม (ควรใช้เมื่อเป็น Constants เท่านั้น)
5. **การส่งค่าแบบ "ถ่ายเอกสาร" (Pass by Value):** สาธิตการส่งสำเนาข้อมูลข้ามฟังก์ชัน ที่ไม่สามารถเปลี่ยนแปลงตัวแปรต้นฉบับได้
6. **ไขความลับ Pointer:**
   - **การดูที่อยู่หน่วยความจำ (Print Address):** พาไปแอบดู "เลขห้อง" ใน RAM ของคอมพิวเตอร์ด้วยการใช้ `%p`
   - `&` (Address-of): สัญลักษณ์สำหรับใช้ "ขอเลขห้อง" (ที่อยู่หน่วยความจำ)
   - `*` (Dereference): สัญลักษณ์ที่เป็นกุญแจสำหรับวาร์ปไปไขเปิดกล่องเพื่อดูหรือแก้ไขข้อมูล
7. **บทส่งท้าย (Intro to Linked List):** ก้าวแรกสู่โครงสร้างข้อมูลแบบขบวนรถไฟที่เพิ่ม-ลดขนาดได้อย่างอิสระ

## 📂 โครงสร้างของโปรเจกต์

**หมวดหมู่: Function**
- `x+5.c` - พื้นฐานการสร้างฟังก์ชันจากสมการคณิตศาสตร์
- `pros_of_func.c` - เปรียบเทียบให้เห็นข้อดีของโค้ดที่ใช้ฟังก์ชัน vs ไม่ใช้ฟังก์ชัน
- `return_vs_void.c` - ตัวอย่างฟังก์ชันแบบมีค่า Return และแบบ Void
- `param_vs_argu.c` - จุดสังเกตความแตกต่างระหว่าง Parameter และ Argument
- `global_var.c` - วิธีใช้งาน Global Variable ที่ปลอดภัย (Constants)
- `pass_by_value.c` - โชว์ความล้มเหลวในการแก้ไขค่าตัวแปรข้ามฟังก์ชันด้วยการถ่ายเอกสาร

**หมวดหมู่: Pointer**
- `print_address.c` - ทดลองใช้ `%p` เพื่อปริ้นท์ที่อยู่ของหน่วยความจำออกมาดู
- `address_vs_deref.c` - ทำความรู้จัก `&` และ `*` พร้อมจำลองการทะลวง Memory ไปแก้ไขค่า
